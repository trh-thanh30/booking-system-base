# Grill With Docs

Use this before large or ambiguous work.

1. Read `CONTEXT.md`, related ADRs, and the closest app/package files.
2. Ask only questions that cannot be answered from the repo.
3. Walk the decision tree until dependencies between choices are resolved.
4. Update `CONTEXT.md` when a durable term or rule appears.
5. Create an ADR when the decision affects architecture, ownership, or long-term constraints.
